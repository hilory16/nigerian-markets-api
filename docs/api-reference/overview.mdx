# API Reference

The API surface is intentionally small.

## Endpoints

| Method | Endpoint | Purpose |
| --- | --- | --- |
| `GET` | `/api` | API index and discovery |
| `GET` | `/api/states` | List all states |
| `GET` | `/api/states/{state}` | Get one state and its LGAs |
| `GET` | `/api/lgas/{lga}` | Get one LGA, its parent state, and its markets |
| `GET` | `/api/markets` | List markets with pagination |
| `GET` | `/api/search` | Search markets, LGAs, and states |
| `POST` | `/api/contribute` | Submit a market suggestion |

## OpenAPI source of truth

The contract lives in [`docs/openapi/openapi.yaml`](../openapi/openapi.yaml). That file is the source of truth for future generated API reference pages and SDK workflows.

## Parameter notes

- `limit` is clamped to the `1-100` range
- `offset` starts at `0`
- `order` accepts `asc` or `desc`
- `q` is required for search and limited to 100 characters

## Error model

Errors use a consistent structure:

```json
{
  "success": false,
  "error": {
    "message": "State not found",
    "code": "NOT_FOUND"
  }
}
```
